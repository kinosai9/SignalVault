# SignalVault 用户使用手册

> 适用版本：Release Engineering 封板基线（2026-07-15）  
> 使用对象：需要持续整理投资研究资料、但不希望直接操作数据库或代码的个人用户  
> 安全边界：SignalVault 用于整理来源、观点、风险和待验证信号，不提供买入、卖出或持有建议。

## 1. 先理解一件事

SignalVault 不是“问一个问题就给答案”的聊天工具。它解决的是一条长期研究链路：

```text
外部资料
  → 预览与确认
  → 分析或归档
  → 报告、观点、信号、实体
  → 原文证据
  → 关注与后续跟踪
```

日常使用时不需要记住技术模块。只要按四个问题行动：

1. **今天有什么变化？** 去“变化雷达”或“我的关注”。
2. **有新资料要放进来？** 去“导入中心”。
3. **过去是否出现过相关观点？** 去“知识搜索”。
4. **哪里失败了、下一步做什么？** 去“任务与诊断”。

## 2. 第一次使用

### 2.1 安装与启动

以下步骤通常只需要做一次。Windows PowerShell：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -e “.[dev]”
python -m signalvault serve
```

浏览器打开：

```text
http://127.0.0.1:8000/
```

SignalVault 默认只监听本机 `127.0.0.1`，没有登录鉴权，不要直接暴露到公网。

> **不再需要手工复制 `.env.example`。** 配置通过 Web Console 完成，自动保存。

### 2.2 配置 AI 服务

打开左侧导航 → **系统与集成** → **AI 服务**：

1. Provider 选择 `openai-compatible`。
2. 填写 Base URL（例如 `https://api.openai.com/v1`）和 Model 名称。
3. 输入 API Key，点击「保存 Key」。
4. 点击「测试连接」确认配置正确。

Mock 模式（默认）可用于验证工程链路，但不具备真实分析能力。页面会显示当前状态：Mock / 配置不完整 / 已配置尚未验证 / 正常 / 验证失败。

API Key 加密存储，不进日志、不进页面源码、不进诊断包。

### 2.3 配置 Obsidian 知识库（可选）

Obsidian 是可选集成。未配置时所有核心功能不受影响。

打开 **系统与集成** → **Obsidian**：

1. 启用集成。
2. 输入 Vault 路径（必须是已存在的目录的完整绝对路径）。
3. 系统自动验证路径可读可写。
4. 查看「初始化预览」了解将要创建的目录和文件。
5. 点击「初始化 Vault」创建 SignalVault 结构。
6. 如有缺失内容，可用「修复」补齐（不会覆盖已有文件）。

建议创建专用 Obsidian Vault，然后用 Obsidian 打开该文件夹。

> SQLite 是 SignalVault 主数据源；Obsidian 是可选的知识库浏览、导出和人工编辑层。

## 3. 页面地图

左侧导航按用户任务分成三组：

| 导航 | 什么时候用 | 页面重点 |
|---|---|---|
| 变化雷达 | 每天第一次打开 | 关注点变化、待处理来源、诊断状态、今日动作 |
| 我的关注 | 复核重点标的或主题 | 新证据、观察项、跟踪状态 |
| 知识搜索 | 查历史观点和证据 | 报告、观点、信号、实体及来源筛选 |
| 导入中心 | 有新资料时 | 按来源类型选择正确入口 |
| 信息源工作台 | 管理来源与待办 | 各来源状态、待确认项、失败项 |
| 报告库 | 阅读和筛选研究结果 | 报告列表、来源、日期和详情 |
| 任务与诊断 | 等待任务或处理失败 | 系统健康、恢复建议、操作日志、任务进度 |
| API | 需要程序集成时 | 本地只读 API 说明 |

“YouTube、网页、固定源、文件/PDF、知识星球”保留在”系统与集成”区，适合直接进入某一种来源的管理页。

| 导航 | 什么时候用 | 页面重点 |
|---|---|---|
| 系统与集成 | 配置 AI、Obsidian 或查看系统状态 | AI 服务配置、Obsidian 集成、数据与系统、诊断与关于 |
| AI 服务 | 配置 LLM | Provider、API Key、测试连接、Mock 状态 |
| Obsidian | 配置可选知识库 | Vault 路径、初始化/修复、写入测试 |
| 数据与系统 | 查看系统信息 | 应用版本、路径、数据库、服务状态（只读） |
| 诊断与关于 | 查看环境与诊断入口 | 系统环境、健康状态、隐私说明、许可证、免责声明 |

## 4. 推荐的日常工作流

### 场景 A：每天 10 分钟看研究变化

目标：先处理真正影响研究判断的变化，不从报告列表逐条翻起。

1. 打开左侧“变化雷达”。
2. 先看顶部雷达卡：关注点、待处理来源、诊断状态是否有变化。
3. 看“今天先处理”，优先处理失败项、待确认导入和需要复核的证据。
4. 打开“我的关注”，按关注对象查看新证据和观察项。
5. 对不确定的内容点击相关报告，回到观点、风险和原文引用。
6. 没有新动作时结束，不需要为了“清空页面”处理低价值事项。

### 场景 B：导入一条新的研究资料

目标：让系统先判断资料类型、质量和重复情况，用户只在写入前做一次确认。

1. 打开“导入中心”。
2. 按资料本身选择 YouTube、知识星球、网页链接、固定信息源或文件/PDF。
3. 先生成预览，不会立刻写入 Vault。
4. 检查标题、正文摘要、解析质量、重复冲突和系统推荐动作。
5. 确认后再归档或进入分析；不需要的内容选择“跳过”。
6. 回到“信息源工作台”确认状态；分析类资料完成后进入报告详情。

### 场景 C：验证一个历史判断

目标：查清“谁在什么来源中说过什么”，而不是只看 AI 摘要。

1. 打开“知识搜索”。
2. 输入公司、产品、技术、行业或观点关键词。
3. 先用“观点/信号/报告/实体”筛选结果层级。
4. 再按 YouTube、知识星球、PDF、本地文本筛选来源。
5. 观点结果重点看方向、逻辑链、风险提示和原文证据。
6. 点击“查看报告”，在报告详情复核上下文。
7. 如果报告有“查看完整原文（中英对照）”，继续打开并按时间戳或页码阅读原材料。

当前 Web 搜索页展示报告、观点、信号和实体。SourceDocument / SourceSegment 已在底层落地，但尚未作为 Web 筛选项开放。

### 场景 D：导入或分析失败

目标：先知道问题发生在哪一层，再决定重试、修复配置或跳过。

1. 打开“任务与诊断”，实际路由是 `/tasks`。
2. 先看系统健康卡，确认是来源、授权、解析、LLM、数据库、Vault 还是搜索/图谱问题。
3. 阅读“建议动作”。系统不会自动执行这些动作，需要用户确认。
4. 在操作日志中找到最近失败记录，查看错误摘要。
5. 在任务列表打开对应任务，查看进度和日志。
6. 需要远程排障时点击“导出诊断包”。诊断包会脱敏，但发送前仍建议人工检查文件内容。

不要只重复点击导入。授权未登录、扫描型 PDF、LLM 配置错误等问题，重试不会自动消失。

## 5. 各类资料怎么导入

### 5.1 YouTube：单个视频快速整理

适合临时收到一个明确视频链接：

1. 打开 `/content/new`。
2. 粘贴 YouTube 链接。
3. 填写关注方向，多个方向用逗号分隔。
4. 选择“标准”或“深度”。深度模式耗时和 LLM 成本更高。
5. 选择“整理进知识库”或“仅生成报告”。
6. 首次验证链接和字幕时，可勾选“测试模式”。
7. 点击“开始整理”，到“任务与诊断”查看进度。

测试模式成功只说明链接、字幕和工程链路可用，不说明观点质量可用。

### 5.2 YouTube：长期跟踪频道

适合持续关注同一批播客或频道：

1. 从“导入中心”选择“YouTube 视频”。
2. 在频道页添加频道 URL、名称、标签、优先级和默认关注点。
3. 点击刷新，获取候选视频。
4. 进入频道视频页，筛选新视频。
5. 对有价值的视频执行导入/分析；无关视频标记跳过。
6. 分析完成后从报告或任务页查看结果。

长期来源优先走频道管理，不要每次重复粘贴单视频链接。

### 5.3 PDF 研报

Web 入口：`文件 / PDF`。

1. 上传不超过 20 MB 的 `.pdf`。
2. 系统逐页提取文本并显示质量、文本长度、摘要和冲突。
3. 确认后进入分析，完成后跳到报告详情。
4. 在观点证据中核对 PDF 页码。

边界：

- 文本型 PDF 可以直接处理。
- 扫描图片型 PDF 会被标记为需要 OCR，当前不会自动调用外部 OCR。
- Web PDF 分析当前使用 mock provider。需要真实 LLM、指定关注点或批量处理时使用：

```powershell
python -m signalvault pdf analyze report.pdf --no-mock --focus "AI投资,美股"
```

### 5.4 文本、Markdown 和 HTML 文件

1. 在“文件 / PDF”上传 `.md`、`.txt`、`.html` 或 `.htm`，上限 5 MB。
2. 检查提取摘要、编码、质量和重复冲突。
3. 点击“确认归档”，内容进入 SourceArchive。

普通文本资料默认只归档，不自动生成投资报告，这是为了避免把资料保存误当成观点抽取。

### 5.5 普通网页

1. 在“导入中心”选择“网页链接”。
2. 粘贴 URL 并生成预览。
3. 检查正文摘要、解析质量、标题/URL/内容哈希冲突。
4. 按系统推荐选择归档、关联精读笔记、覆盖或跳过。

网页正文抽取是 best-effort。遇到登录墙、复杂脚本页面或正文很短时，应先检查预览，不要直接确认。

### 5.6 固定信息源

适合会持续新增内容的网页来源，目前重点支持 All-In 中文笔记：

1. 打开“固定源”，选择添加来源。
2. 输入 URL，先做 profiling。
3. 系统确认可跟踪后创建来源。
4. 点击刷新，发现新条目。
5. 批量选择条目并确认导入。

如果系统提示该 URL 不支持跟踪，改用“网页链接”做单次导入。

### 5.7 知识星球

前置条件：已经在官方 App/Web 订阅目标星球，并按外部工具说明安装、登录 `zsxq` 或兼容的 `zsxq-cli`。

1. 打开“知识星球”。
2. 检查“CLI 状态”和“登录状态”。未登录时先在终端执行 `zsxq auth login`。
3. 点击“刷新星球列表”。
4. 选择已授权星球，设置读取最近主题数量，点击“同步最新主题”。
5. 已知 topic ID 时，可展开高级操作，选择“只导入”或“导入并分析”。
6. 分析时可填写关注点、深度和模型模式。

SignalVault 只读取已经授权的内容，不搜索公开星球，不订阅、发布、评论、点赞、删除或管理成员。

## 6. 如何阅读一份报告

报告详情不是最终答案页，建议按以下顺序阅读：

1. **来源与元数据**：确认内容来自哪个视频、PDF 或知识星球主题。
2. **核心观点**：看方向、逻辑链和证据强度，不只看结论句。
3. **风险提示**：确认原观点的条件和反例是否被保留。
4. **跟踪信号**：区分“已经发生的事实”和“未来需要验证的条件”。
5. **原文证据**：核对 `source_quote`、时间戳或 PDF 页码。
6. **完整原文**：需要上下文时打开“查看完整原文（中英对照）”。

外文原文页可以触发翻译。翻译是派生内容，不覆盖原文；翻译失败的片段会单独标记。

报告页的“同步到知识库”会把报告导出到 Obsidian，并更新研究摘要与关注内容。只想先审阅报告时，不必立即同步。

## 7. 我的关注怎么用

1. 打开“我的关注”查看当前重点对象的新证据和观察项。
2. 通过“Watchlist 设置”增加或移除公司、主题等关注对象。
3. 新资料分析后，系统会按关注对象匹配证据。
4. 从关注项进入报告和原文，确认它是否真的改变原有判断。

关注列表的作用是压缩每天需要阅读的内容，不是生成买卖信号。建议只保留真正会持续复核的对象。

## 8. 常见状态与处理方式

| 状态 | 含义 | 建议动作 |
|---|---|---|
| 待处理 | 尚未开始预览或分析 | 打开对应来源继续 |
| 待确认 | 预览已完成，尚未写入 | 检查摘要、冲突和推荐动作 |
| 新发现 | 固定源发现新条目 | 选择导入或跳过 |
| 已发现 | 已经见过该条目 | 通常不需要重复处理 |
| 已入库 | 已完成归档或分析 | 去报告库或知识搜索查看 |
| 已跳过 | 用户主动不处理 | 无需操作，可按需重新导入 |
| 解析退化 | 提取到内容但质量较低 | 先核对预览，再决定是否保留 |
| 需人工确认 | 系统不能安全自动判断 | 查看审核项或诊断建议 |
| 失败 | 当前操作未完成 | 去“任务与诊断”查具体层级和恢复动作 |

## 9. 系统与集成配置

日常使用中偶尔需要查看或调整系统配置。所有配置页面从左侧导航 → **系统与集成** 进入。

### 9.1 概览

`/settings` 一览 AI 服务状态、Obsidian 集成状态、系统信息和诊断状态。适合作为"系统体检"入口。

### 9.2 AI 服务

`/settings/ai` 配置 LLM Provider、Model、Base URL 和 API Key。Key 加密存储，不回显。配置变更后旧验证自动失效，需要重新测试连接。

常见问题：
- **"已配置，尚未验证"**：点击测试连接
- **"配置被环境变量覆盖"**：页面值不生效，去 `.env` 中移除对应变量
- **Mock 模式**：不产生真实分析，仅验证工程链路

### 9.3 Obsidian 集成

`/settings/obsidian` 可选配置 Obsidian Vault。详细步骤见 2.3 节。

### 9.4 数据与系统

`/settings/system` 只读展示应用版本、路径（含备份/清理建议）、数据库状态和服务信息。

### 9.5 诊断与关于

`/settings/about` 展示系统环境、隐私说明和免责声明。"任务与诊断"入口链接到 `/tasks`。

诊断功能主入口始终是 `/tasks`，不是 `/settings/about`。需要导出诊断包时从 `/tasks` 操作。

### 9.6 配置存储

- **SecretStore**：API Key 单独加密存储（`secrets` 文件），不进 `config.toml`
- **config.toml**：普通配置（Provider、Model、路径等）
- **.env**：开发者/高级用户的覆盖方式。普通用户无需编辑 `.env`

## 10. CLI 什么时候更合适

日常单条操作优先使用 Web。以下情况使用 CLI 更直接：

- 批量处理或脚本自动化
- 真实 LLM PDF 分析和关注点控制
- 全量知识图谱重建或 JSON 导出
- Vault lint、审核队列和高级 Obsidian 维护
- MCP Server 启动
- 诊断摘要和诊断包自动化

常用入口：

```powershell
python -m signalvault --help
python -m signalvault search "NVIDIA"
python -m signalvault doctor
python -m signalvault diagnostics bundle
python -m signalvault mcp-serve
```

完整命令见 `README.md` 和 `docs/DEV_GUIDE.md`。

## 10. 数据、隐私与成本

- 数据库、报告、原文和 Vault 默认保存在本机。
- 知识星球、上传文件等私有来源默认按本地材料处理。
- 诊断日志不应记录 API Key、cookie、完整原文或报告全文。
- 真实 LLM 会把分析所需文本发送到配置的模型服务，并产生调用费用。
- 长视频会自动分块；N 个块通常意味着多次事实抽取调用，应在运行前考虑成本。
- MCP、API 和 Web 服务都以本地只读或本地操作为主，不要暴露到不受信任网络。

## 11. 当前明确边界

- 不提供投资建议。
- 不做登录鉴权、团队协作或云同步。
- 不做自动定时抓取。
- 不做 RAG、向量数据库或 AI 问答。
- 不自动 OCR 扫描型 PDF。
- 不支持 Office 文件导入。
- 不对知识星球执行任何写入动作。
- Web 搜索暂不直接展示原文材料/片段结果。

## 12. 遇到问题先看这里

**页面打不开**  
确认运行 `python -m signalvault serve`，再访问 `http://127.0.0.1:8000/`。

**提示 Vault 未配置或目录不完整**  
进入 `/setup/vault` 重新选择目录，或使用页面提供的修复动作。

**YouTube 没有字幕**  
先在测试模式验证；确认视频有可用字幕。主适配器失败时可进一步检查 yt-dlp 备用路径和任务日志。

**PDF 没有生成报告**  
检查是否为扫描件、文本量是否过少，再到“任务与诊断”查看 `pdf_needs_ocr` 或质量问题。

**知识星球列表为空**  
确认外部 CLI 已安装并登录，然后在知识星球页点击“刷新星球列表”。

**搜索不到刚导入的资料**  
普通网页/文本可能只归档为 SourceArchive，不一定生成报告、观点或信号；当前 Web 搜索也未开放原文材料结果。

**真实分析失败**  
检查 `.env` 的 provider、base URL、model 和 API Key；不要把密钥粘贴到诊断信息中。

**需要他人协助排障**  
在“任务与诊断”导出诊断包，人工检查后再发送。不要发送 `.env`、数据库或完整付费原文。
